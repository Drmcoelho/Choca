# M30_ASSESSMENT_SPEC.md — Exame global de domínio · PERFUNDE · CHOCA

Este documento especifica o M30.

M30 não é mais um módulo de conteúdo. É a prova global, holística e robusta do braço: 100 questões de dificuldade crescente, cobrindo todo o currículo em ordem embaralhada de módulos, com gabarito explicativo denso e controle contra padrões artificiais de resposta.

---

## 1. Identidade do M30

```text
M30 · Revisão global · exame de domínio
```

Função:

```text
verificar se o aluno decompõe fisiologia, reconhece armadilhas, integra módulos e evita rótulo decorado
```

Não é OSCE, não é capstone clínico narrativo, não é banco solto de questões. É exame estruturado de domínio.

Relação com M29:

```text
M29 = capstone integrado narrativo / caso-síntese
M30 = exame global de domínio / avaliação psicométrica
```

---

## 2. Estrutura macro

Total:

```text
100 questões
4 quartos de 25 questões
```

Os quartos têm dificuldade crescente:

```text
Q1 · questões 1–25   → reconhecimento e decomposição básica
Q2 · questões 26–50  → aplicação e contraste entre módulos
Q3 · questões 51–75  → integração, conflito de variáveis, pegadinhas fisiológicas
Q4 · questões 76–100 → alta dificuldade, choques mistos, exceções, leitura contraintuitiva
```

A ordem dos módulos dentro de cada quarto deve ser embaralhada. O exame não deve seguir a sequência M0, M1, M2… porque isso facilita por contexto, não por domínio.

---

## 3. Ordem aleatória com dificuldade crescente

Regra central:

```text
ordem de módulos = aleatória
nível cognitivo = crescente
```

Exemplo:

```text
Q1 pode alternar M1, M9, M14, M3, M12...
Q2 pode misturar M20, M5, M8, M16...
Q3 pode cruzar M21 com M13, M17 com M24, M15 com M1...
Q4 pode exigir M23/M29/M28/M26 com múltiplos termos simultâneos
```

O aluno não deve prever o próximo tema pelo número da questão.

---

## 4. Distribuição por letras

O exame tem alternativas A, B, C e D.

Regras:

1. Nenhuma letra deve ser dominante de forma óbvia.
2. Nenhuma letra precisa ter exatamente 25 respostas.
3. Cada letra deve aparecer como correta entre 15 e 35 vezes no total.
4. A distribuição deve ser assimétrica o bastante para não parecer fabricada.
5. A distribuição por quarto também não deve criar padrão.

Permitido:

```text
A = 16, B = 34, C = 17, D = 33
A = 18, B = 32, C = 19, D = 31
A = 21, B = 27, C = 24, D = 28
A = 30, B = 18, C = 34, D = 18
```

Proibido:

```text
A = 25, B = 25, C = 25, D = 25  # artificial demais
A = 40, B = 20, C = 20, D = 20  # fora da faixa
A = 10, B = 30, C = 30, D = 30  # fora da faixa
```

Observação: não há obrigação de pares somarem 50. A+B pode somar 40, 45, 55, 60 etc. O único limite rígido é cada alternativa ficar entre 15 e 35 no total.

---

## 5. Proibição de padrões de gabarito

O gabarito não pode conter pistas artificiais.

Proibido:

```text
sequências longas da mesma letra
alternância regular A-B-C-D
alternância em pares A-A-B-B-C-C-D-D
letra correta previsível por posição
letra correta previsível por extensão do texto
opção mais longa ser frequentemente correta
opção com número maior ser frequentemente correta
opção mais técnica ser sempre correta
“todas as anteriores” como atalho
“nenhuma das anteriores” como fuga
```

Regra prática:

```text
a alternativa correta deve ser correta por fisiologia, não por forma
```

---

## 6. Construção das alternativas

Cada questão deve ter 4 alternativas plausíveis.

A alternativa correta deve ser precisa, mas não pode se destacar por estilo. Os distratores devem representar erros reais de raciocínio.

Cada distrator deve mapear um erro:

```text
A/B/C/D → conceito errado → por que parece plausível → por que falha
```

Tipos desejados de distrator:

```text
confundir número com mecanismo
confundir conteúdo com saturação
confundir PAM com perfusão
confundir responsivo com tolerante
confundir distributivo com séptico
confundir SvO2 alta com boa extração
confundir hipovolemia com qualquer hipotensão
confundir VD com VE
confundir pós-carga com PA
confundir procedimento com indicação
confundir mecanismo farmacológico com dose
```

## 7. Níveis de dificuldade

### Q1 — Reconhecimento

O aluno identifica termos básicos.

Exemplos:

```text
CaO2 depende principalmente de Hb e saturação, não de PaO2 isolada.
PAM é produto/dependente de DC e RVS.
Hipovolêmico quebra retorno venoso.
```

### Q2 — Aplicação

O aluno aplica equações e diferencia fenômenos próximos.

Exemplos:

```text
mesma PAM com DC/RVS diferentes
lactato alto com SpO2 normal
volume responsivo mas não tolerante
```

### Q3 — Integração

O aluno cruza módulos.

Exemplos:

```text
hemorragia = volume + conteúdo
VD falha por pós-carga pulmonar
séptico = macro talvez ok, micro/mito quebradas
```

### Q4 — Alta dificuldade

O aluno interpreta exceções, simultaneidade e armadilhas.

Exemplos:

```text
choque misto
SvO2 paradoxal
PAM normal com lactato alto
neurogênico sem taquicardia
anafilático com eixo respiratório e leak
pressor melhora macro, não necessariamente micro
```

## 8. Cobertura curricular

O exame deve cobrir todos os blocos.

Distribuição sugerida, não rígida:

```text
M0–M2    fundamentos          10–15 questões
M3–M8    determinantes        20–25 questões
M9–M13   leitura/beira-leito   20–25 questões
M14–M22  choques              25–35 questões
M23–M29  integração           10–20 questões
```

M30 avalia todo o braço. Não deve virar prova só de choque séptico, só de drogas ou só de fórmulas.

---

## 9. Gabarito ultra-robusto

Cada questão deve ter gabarito com cinco camadas:

```text
1. resposta correta
2. justificativa fisiológica
3. por que cada alternativa errada é tentadora
4. qual conceito/módulo a questão avalia
5. qual armadilha cognitiva foi testada
```

Formato ideal:

```text
Gabarito: C
Conceito: M9 · PAM = DC × RVS
Por que C: decomposição mostra que a PAM igual pode vir de DC alto/RVS baixa ou DC baixo/RVS alta.
Por que A erra: confunde PAM com perfusão.
Por que B erra: assume que RVS sempre sobe no choque.
Por que D erra: ignora que DC pode estar alto no distributivo.
Armadilha: número-resultante ≠ mecanismo.
```

## 11. Validador do M30

O M30 deve ter validador próprio.

Validações obrigatórias:

```text
100 questões exatas
4 quartos de 25
cada questão com A/B/C/D
cada questão com uma única correta
cada letra correta entre 15 e 35 vezes no total
distribuição por quarto sem padrão grosseiro
sem sequência longa de mesma letra
sem repetição mecânica de módulos
metadados presentes
rationale completo para A/B/C/D
nenhuma questão com dose, prescrição ou conduta de paciente real
```

Validações desejáveis:

```text
detecção de alternativa correta sistematicamente mais longa
detecção de “maior número sempre correto”
detecção de termos absolutos repetidos
checagem de cobertura mínima por bloco
checagem de progressão de dificuldade por quarto
```

---

## 12. Forma de construção

O M30 deve ser construído em quatro entregas:

```text
Parte 1 → questões 1–25
Parte 2 → questões 26–50
Parte 3 → questões 51–75
Parte 4 → questões 76–100
```

Cada parte deve passar por auditoria antes da próxima.

Após as quatro partes:

```text
1. auditar distribuição A/B/C/D
2. auditar cobertura de módulos
3. auditar dificuldade crescente
4. auditar ausência de padrões
5. auditar gabaritos
6. publicar HTML final
```

---

## 13. Critério de excelência do M30

O M30 está excelente quando:

```text
não há padrão de resposta explorável
não há pista formal de alternativa correta
cada distrator ensina uma armadilha
cada gabarito explica o mecanismo
os módulos aparecem fora de ordem previsível
a dificuldade cresce por raciocínio, não por enunciado longo
os casos integram fisiologia sem virar conduta
```

O aluno que passa no M30 não decorou o curso. Ele internalizou o mapa causal.
